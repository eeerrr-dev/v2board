# NNNN. 标题(动词短语,写清决策对象)

- 状态:提议 | 已采纳 | 已取代(由 NNNN)
- 日期:YYYY-MM-DD

## 背景(Context)

当时面对的问题与约束:规模、运维能力、既有契约、安全边界。写事实,不写
愿望。

## 决策(Decision)

选了什么,一句话可复述。明确列出**被拒绝的替代方案**及拒绝理由 —— 没有
被拒绝项的 ADR 通常说明决策还没想清楚。

## 后果(Consequences)

- 得到了什么(简化、保证、门禁)。
- 付出了什么(限制、单点、维护成本)。诚实写代价,后果不全是好消息。

## 证据

指向仓库内落实此决策的文档与代码(相对路径)。
